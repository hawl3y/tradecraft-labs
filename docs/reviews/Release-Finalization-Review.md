## Vercel Readiness

**Required environment variables — set in Vercel before deploying:**

| Variable | Value | Where it's used |
|---|---|---|
| `NEXT_PUBLIC_SITE_URL` | `https://tradecraftlabs.com` | `metadataBase` (all OG/Twitter tags), sitemap URLs, OG image domain display. Without this, all canonical URLs and social cards point to the Vercel fallback domain. |
| `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` | `tradecraftlabs.com` | Analytics script `data-domain` attribute. If unset, no analytics injected — platform still functions normally. |

**Deployment configuration:**
- No `vercel.json` needed — Vercel auto-detects Next.js 15
- Set Node.js version to **20.x** in Vercel → Settings → Build & Development Settings → Node.js Version
- `package-lock.json` is present — Vercel will use `npm ci` automatically
- Build command: `next build` (Vercel default)
- Output directory: `.next` (Vercel default)

**Domain configuration:**
- Add custom domain in Vercel → Settings → Domains
- Set `NEXT_PUBLIC_SITE_URL` to match the domain after DNS is configured
- Until DNS resolves, set `NEXT_PUBLIC_SITE_URL` to the Vercel subdomain (`tradecraftlabs.vercel.app`) to prevent broken OG tags during the interim period

**Analytics configuration:**
- Register domain at plausible.io first
- Set `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` to the exact registered domain
- Plausible dashboard will show real-time hits once script loads on production
