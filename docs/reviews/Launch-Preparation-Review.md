## 1. Analytics

### Implementation

Plausible analytics is configured via a server component that conditionally renders the script:

- **Component:** `components/analytics/PlausibleScript.tsx`
- **Strategy:** `afterInteractive` via `next/script` — loads after page hydration, does not block rendering
- **Control:** `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` environment variable
  - Set → analytics script is injected with `data-domain={domain}`
  - Unset → no script is rendered (safe for local development)
- **Placement:** `app/layout.tsx`, rendered after `</div>` of main content and before `</body>`

### Required configuration

Before deploying, set in the Vercel project environment variables:

```
NEXT_PUBLIC_PLAUSIBLE_DOMAIN=tradecraftlabs.com
```

The value must match the domain registered in the Plausible account exactly (no `https://`, no trailing slash, no path).

### Plausible setup (if not done yet)

1. Create an account at plausible.io
2. Add site: `tradecraftlabs.com`
3. Copy the domain value to the Vercel env var
4. Redeploy — the script will be included in the next build

**Local development:** Do not set `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` in `.env.local`. Analytics script will be absent during development, which is correct behavior.

## 5. Accessibility Validation

### Skip-to-content link

**Implementation:** `app/layout.tsx` — first element in `<body>` before `<Header />`

```html
<a href="#main-content" class="sr-only focus:not-sr-only ...">
  Skip to main content
</a>
```

- Visually hidden (`sr-only`) until focused
- On focus: appears as navy-background button at top-left (z-index 100, above sticky header)
- Target: `<div id="main-content" tabIndex={-1}>` wrapping all page content
- `tabIndex={-1}` on the target allows programmatic focus without adding it to the natural tab order

**`<main>` note:** The content wrapper uses `<div id="main-content">` rather than `<main>`. For full semantic compliance, `<main>` is the preferred landmark element. Using `<div>` with `id="main-content"` satisfies the skip-to-content anchor but is not a proper `<main>` landmark. This is a minor semantic gap — screen readers using landmark navigation will not find a `<main>` region. Recommended: change `<div id="main-content">` to `<main id="main-content">` in `app/layout.tsx`.
