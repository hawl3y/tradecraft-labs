# Open Graph Assets

## og-default.png

Static PNG copy of the default OG image. Generated from the live route `/opengraph-image`.
Used as a visual reference — the production OG image is served dynamically by Next.js.

**Dimensions:** 1200 × 630 px  
**Format:** PNG  
**Design:** Cream background (#FAFAF8), navy left accent bar, Georgia serif mission statement,
wordmark + "Independent Research" descriptor, domain at bottom.

## How the OG image works

The production OG image is generated by `app/opengraph-image.tsx` using `next/og`
(ImageResponse / Satori). It is statically rendered at build time and served at:

    /opengraph-image

Next.js automatically includes it as `og:image` in all pages that inherit the root layout metadata.

## Updating

To regenerate `og-default.png` after modifying `app/opengraph-image.tsx`:

1. Run the dev server: `npm run dev`
2. Fetch the image: `curl http://localhost:3000/opengraph-image -o public/og/og-default.png`

## Per-page OG images

To add a custom OG image for a specific route, create an `opengraph-image.tsx` inside
that route's directory. Example: `app/research/[project]/opengraph-image.tsx`.
